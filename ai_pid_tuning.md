# AI 调参指南 — 单车侧倾平衡串级 PID

> **文档用途**：本文件是给 **AI 助手** 读的"调参工作手册"。
> 用户会按"当前正在调 XX 环 + 串口打印数据 + VOFA+ 波形截图"的形式来提问，
> AI 必须基于本文档给出的**原理、阈值、命令格式**作出诊断，并**直接给出可粘贴的串口命令**。
>
> **最终输出格式要求**：
>
> ```

> 【调整建议】<环名> 的 <Kp/Ki/Kd/target/limit> 应 <加大/减小> <量级>，理由 <依据>。
> 【命令】change-<环名>(Kp,Ki,Kd,target,limit)
> ```
>
> 其中**命令行必须是整行可直接发送给无线串口**，不得缺参数，不得添加注释。

## 1. 系统架构概览

**车模类型**：单车模型（two-wheel **自行车式**平衡车），靠前轮舵机打角产生的陀螺进动维持**侧倾**（横滚）平衡。

**控制对象**：通过舵机 PWM 偏移量，控制车体横滚角稳定在 0°（或转向环指定的目标倾角）。

**控制结构**：**三环串级位置式 PID**（外→中→内）。

| 环 | 函数入口 | 周期 | 调度位置 | 输入 | 输出 |
| -- | -------- | ---- | -------- | ---- | ---- |
| **外环**（转向） | `balance_steering_loop(err)` | 20 ms | [isr.c:75-83](../user/isr.c#L75-L83) `CCU61_CH1` | 转向误差 | `target_angle`（目标横滚角） |
| **中环**（角度） | `balance_angle_loop()` | 10 ms | [isr.c:65-73](../user/isr.c#L65-L73) `CCU61_CH0` | `target_angle` − `pitch` | `target_gyro_rate`（目标角速度） |
| **内环**（角速度） | `balance_gyro_loop()` | 2 ms | [isr.c:55-63](../user/isr.c#L55-L63) `CCU60_CH1` | `target_gyro_rate` − `gyro_y_rate` | `servo_output`（舵机 PWM 偏移） |

**最终输出**：`servo_set(mid + servo_output)` 见 [balance_app.c:71](../code/app/balance_app.c#L71)。

## 2. ⚠️ 术语陷阱 — 动手前必读

> 本工程变量命名**与物理含义不对齐**，AI 在响应时必须按以下约定解释：

| 代码变量 | 实际物理量 | 正负约定 | 来源 |
| -------- | --------- | -------- | ---- |
| `pitch` | **横滚角**（左右倾斜，绕车体前后轴） | 左倾为正 / 右倾为负 | [imu_app.c:104-105](../code/app/imu_app.c#L104-L105) |
| `gyro_y_rate` | 横滚角速度（绕同轴角速度） | 方向同 `pitch` | [imu_app.c:109](../code/app/imu_app.c#L109) |
| `roll` | 实际是俯仰角 | **全局禁用！不得在控制逻辑里用** | [CLAUDE.md](../CLAUDE.md) |

**AI 在回复里只能使用"横滚角"这一物理概念描述 `pitch`**，不要写"俯仰角"，避免误导用户。

## 3. 串级 PID 数据流

```
 ┌───────────────────────┐
 │ 外环 steering_pid     │  20 ms
 │ in : steering_error   │
 │ out: target_angle     │  limit ±30°
 └───────────┬───────────┘
             │
             ▼
 ┌───────────────────────┐
 │ 中环 angle_pid        │  10 ms
 │ target: target_angle  │
 │ feedback: pitch       │  ← IMU 实时横滚角
 │ out: target_gyro_rate │  limit ±500°/s
 │ (积分限幅 ±200)       │
 └───────────┬───────────┘
             │
             ▼
 ┌───────────────────────┐
 │ 内环 gyro_pid         │  2 ms（最关键！）
 │ target: target_gyro…  │
 │ feedback: gyro_y_rate │  ← IMU 实时角速度
 │ out: servo_output     │  limit ±15000（但实际受舵机硬件限幅）
 └───────────┬───────────┘
             │
             ▼
   servo_set(mid + servo_output)
   mid=5515  l_max=4015  r_max=7015   ← 机械安全硬限幅
```

### 重要饱和关系（AI 必须知道）

- **舵机机械限幅**：`servo_set` 会把 `mid+servo_output` 再次卡进 `[l_max, r_max] = [4015, 7015]`。
- 因此 `servo_output` 的**有效工作范围只有 ±1500**（mid ± 1500）。
- `gyro_pid.limit = 15000` 并不意味着能真正输出这么多；**当 |servo_output| > 1500 即已饱和**，再加 Kp 也没用。
- **诊断时若看到 `servo_output` 长时间贴在 ±1500 附近**，说明内环已饱和，需要减小 Kp 或排查机械问题（舵机没力/角度限幅太紧）。

## 4. 默认参数与硬件限幅清单

| PID | Kp | Ki | Kd | target | limit | 备注 |
| --- | -- | -- | -- | ------ | ----- | ---- |
| `steering_pid` | **1.0** | 0.0 | 0.0 | 0.0 | 30.0 | 外环，转向暂未启用 |
| `angle_pid` | **30.0** | 0.0 | **1.0** | 0.0 | 500.0 | 中环，积分限幅 ±200 |
| `gyro_pid` | **2.0** | 0.0 | 0.0 | 0.0 | 15000.0 | 内环，只用 P |

### 其他关键常数

| 常量 | 值 | 位置 | 含义 |
| ---- | -- | ---- | ---- |
| `mid` | 5515 | [servo_app.h:16](../code/app/servo_app.h#L16) | 舵机中值 |
| `l_max` | 4015 | [servo_app.h:15](../code/app/servo_app.h#L15) | 舵机左极限 |
| `r_max` | 7015 | [servo_app.h:17](../code/app/servo_app.h#L17) | 舵机右极限 |
| `MOTOR_ROLL_CUTOFF_DEG` | 21° | [motor_app.h:18](../code/app/motor_app.h#L18) | 横滚超此值**进入保护**（电机停） |

### 保护状态影响

进入保护态时：

- 电机被强制设 0（停车）。
- `pid_test` 暂停 5 路波形输出（见 [balance_app.c:99-102](../code/app/balance_app.c#L99-L102)）。
- **若用户贴的波形中段有明显断续、电机骤停，先怀疑是横滚超 21° 触发了保护**，不要误判为 PID 问题。

## 5. PID 算法实现

**算法类型**：**位置式 PID**（[pid_driver.c:166-178](../code/driver/pid/pid_driver.c#L166-L178)）。

```
error      = target − current
integral  += error                      // 累加误差
p_out      = Kp × error
i_out      = Ki × integral
d_out      = Kd × (error − last_error)   // 一阶差分
out        = p_out + i_out + d_out
out        = clamp(out, -limit, +limit)  // 输出限幅
```

### 各项直觉

| 项 | 效果 | 副作用 |
| -- | ---- | ------ |
| **Kp ↑** | 响应更快、跟踪误差更小 | 过大会**振荡/啸叫** |
| **Ki ↑** | 消除稳态误差 | 积分饱和 → **超调、低频震荡** |
| **Kd ↑** | 抑制超调、提高阻尼 | 放大高频噪声 → **毛刺/嗡嗡声** |
| **limit ↓** | 防止执行器饱和 | 太小会削弱控制能力 |

### change 会清空状态

调参命令执行后会调用 `pid_reset`（[uart_app.c:108](../code/app/uart_app.c#L108)），**清空积分项和历史误差**，因此改完参数后需要等车体重新进入稳态再判断。

## 6. VOFA+ 波形通道定义

`pid_test` 每 **10 ms** 通过无线串口发送一行 CSV（[balance_app.c:104-105](../code/app/balance_app.c#L104-L105)）：

```
target_gyro_rate, gyro_y_rate, gyro_pid.error, servo_output, pitch
     CH1              CH2          CH3             CH4        CH5
```

| 通道 | 名称 | 单位 | 作用 | 重点观察环节 |
| ---- | ---- | ---- | ---- | ------------ |
| CH1 | `target_gyro_rate` | °/s | 中环输出 = 内环目标 | 中环整形后的指令 |
| CH2 | `gyro_y_rate` | °/s | 内环反馈（IMU 角速度） | 内环追踪 |
| CH3 | `gyro_pid.error` | °/s | 内环误差 = CH1 − CH2 | 内环好坏的直接证据 |
| CH4 | `servo_output` | PWM 偏移 | 内环输出（舵机增量） | **是否饱和**（±1500） |
| CH5 | `pitch` | ° | 当前横滚角 | 外/中环最终表现 |

### AI 解读套路

- **看内环**：CH1 与 CH2 是否重合？重合好 → 内环 OK。CH2 滞后或震荡 → 调 `gyro_pid`。
- **看中环**：CH5 是否稳定在 `target_angle` 附近？CH5 偏离不回归 → 调 `angle_pid`。
- **看饱和**：CH4 是否贴满 ±1500？贴满 = 饱和，先查机械再谈参数。
- **看噪声**：CH2/CH4 高频毛刺 → Kd 过大或滤波不足。

### 6.1 外环调参专用打印数据（当前约定）

> 当用户明确说"正在调外环 / steering_pid / 航向环"时，AI 不应只按上面的内环 5 通道解读，
> 而应优先按下面这 **5 组关键量**理解外环日志。它们对应的是"圆轨迹是否在按目标转、是否转得够、车体是否倾得够、舵机是否已打满"。
>
> **本节已与用户当前调参口径同步**。如果用户说"我现在外环打印的是这 5 个数据"，AI 默认应理解为下面这 **5 组观测量**：
>
> - `wheel_speed`：决定 1 m 圆对应多大的目标转弯强度
> - `yaw / yaw_err`：看有没有沿圆周转
> - `gyro_z_rate`：看实际转弯角速度是否跟上
> - `target_angle / pitch`：看车体横滚是否达到需要的倾角
> - `servo_output`：看舵机是不是已经打满

**推荐打印顺序**：

```csv
wheel_speed,yaw,yaw_err,gyro_z_rate,target_angle,pitch,servo_output
```

> 注意：口头上说是"5 个打印数据"，但在串口实现里它实际展开为 **7 列 CSV**，因为其中两组是成对观测量：
>
> - `yaw / yaw_err`
> - `target_angle / pitch`

| 组 | 实际打印量 | 单位 | 代码来源 | AI 解读重点 |
| --- | --- | --- | --- | --- |
| 1 | `wheel_speed` | m/s | [motor_app.c:232](../code/app/motor_app.c#L232) | 决定 1 m 圆对应多大的目标转弯强度。速度越高，需要的目标偏航角速度和目标倾角越大。 |
| 2 | `yaw`, `yaw_err` | ° | `yaw` 见 [imu_app.c:107](../code/app/imu_app.c#L107)，`yaw_err` 由上层按 `yaw_ref - yaw` 计算 | 看车是否真的沿圆周转，以及当前航向相对目标圆轨迹差了多少。 |
| 3 | `gyro_z_rate` | °/s | [imu_app.c:110](../code/app/imu_app.c#L110) | 看实际转弯角速度是否跟上目标转弯强度。外环是否"转得动"，首先看这个量。 |
| 4 | `target_angle`, `pitch` | ° | `target_angle` 见 [balance_app.c:19](../code/app/balance_app.c#L19)，`pitch` 见 [imu_app.c:104](../code/app/imu_app.c#L104) | 看外环要求的目标横滚角是否被车体真正达到。若 `target_angle` 已很大但 `pitch` 跟不上，问题多半不在外环本身。 |
| 5 | `servo_output` | PWM 偏移 | [balance_app.c:21](../code/app/balance_app.c#L21) | 看舵机是不是已经打满。若长期贴近 ±1500，则优先判定执行器饱和，而不是继续无脑加外环 Kp。 |

### 6.2 外环日志的专用判读规则

- `wheel_speed` 上去了，但 `gyro_z_rate` 很小：说明车速有了，但实际转弯角速度没建立起来，优先查 `target_angle` 是否太小、舵机是否饱和、横滚是否跟不上。
- `yaw_err` 长时间同号且越来越大：说明车没有按目标圆轨迹持续收敛，外环控制强度不足或方向可能错了。
- `target_angle` 已经很大，但 `pitch` 还是明显偏小：说明中环/内环/机械执行链跟不上，不能只调 `steering_pid`。
- `servo_output` 长时间贴近 ±1500：说明舵机有效输出已饱和，再增大外环输入或 Kp 通常只会让日志更难看。
- `yaw` 只适合看短时间趋势；如果用户连续绕圈时间很长，AI 必须警惕 `yaw` 在 `[-180, 180]` 包角处跳变，不要把跳变误判成外环失稳。

## 7. 串口调参命令协议

来源：[uart_app.c:116-175](../code/app/uart_app.c#L116-L175)。行结束符 `\r` 或 `\n`。

### 查看当前参数

```
check-steering_pid
check-angle_pid
check-gyro_pid
```

回复：`[OK] <name>: Kp,Ki,Kd,target,limit`

### 修改参数（核心命令！）

```
change-steering_pid(Kp,Ki,Kd,target,limit)
change-angle_pid(Kp,Ki,Kd,target,limit)
change-gyro_pid(Kp,Ki,Kd,target,limit)
```

- **必须 5 个参数**，逗号分隔，括号包裹。
- 参数顺序固定：`Kp, Ki, Kd, target, limit`。
- 执行后自动 `pid_reset` 清空积分。
- 成功：`[OK] <name> updated`；失败：**静默丢弃**（无回复即格式错）。

### 紧急停车

```
stop
```

回复 `[OK] motor stopped`，进入 4 秒锁定期（[motor_app.c:21](../code/app/motor_app.c#L21)）。

### AI 输出命令时的硬性要求

1. **永远只输出整行、可粘贴的命令**，不要写 `change-angle_pid(Kp=…)` 这种变体。
2. **所有 5 个参数必须写齐**，哪怕只改一个——另 4 个也必须显式填回原值。
3. 如果不确定原参数，**先建议用户发 `check-xxx_pid` 回读**，再给 change 命令。
4. 调参数值保留小数点后 2~3 位即可（例如 `2.50`、`0.005`），不要给 `2.499999`。

## 8. 调参方法论：先内后外

### 铁律

**顺序：gyro_pid（内环） → angle_pid（中环） → steering_pid（外环）。**

内环没调好，外环数据没有参考价值。

### 8.1 调参速度选择（前置必读）

**为什么必须跑起来调？**
本项目是**单车模型**（自行车式平衡车），靠**前轮舵机打角产生的陀螺进动**维持侧倾平衡——**速度越低，进动力矩越小，车越难立住**。静止或低速下舵机打再多角也没有回正力矩，调出来的参数是假的。

**速度单位与换算**：

- 代码中速度命令单位是 **RPM**，反馈通道 `motor_rpm` 同单位，见 [motor_app.c:231-232](../code/app/motor_app.c#L231-L232)。
- 换算到轮速：`wheel_speed (m/s) = rpm × WHEEL_DIAMETER × π / 60 ≈ rpm × 0.00335`，轮径 64 mm，常数见 [motor_app.h:14-16](../code/app/motor_app.h#L14-L16)。
- 限幅上限 ±3000 rpm（[motor_app.c:176-177](../code/app/motor_app.c#L176-L177)），**调参阶段严禁超过 1000 rpm**。

**各阶段速度建议表**：

| 阶段 | 速度 (rpm) | 轮速 (m/s) | 说明 |
| ---- | ---------- | ---------- | ---- |
| ① 内环首次试探 | **300** | ≈ 1.00 | 刚够产生进动，倒了也损失小；舵机余量最大 |
| ② 内环倍增找 Kp | **300~400** | 1.00~1.34 | 正式找 Kp 临界点 |
| ③ 中环起步 | **400** | ≈ 1.34 | [cpu0_main.c:57](../user/cpu0_main.c#L57) 默认值，作者经验速度 |
| ④ 中环精调 | **400~500** | 1.34~1.68 | 接近实车工况 |
| ⑤ 高速复核 | **600~800** | 2.01~2.68 | 验证参数在更高速下不发散 |
| ⑥ 目标巡航 | **实际业务速度** | — | 最终验收 |

**速度调参铁律**：

1. **由低到高，逐级复核**。
   低速调好的参数**不一定**在高速还稳；但高速调的参数**一定**在低速先不稳。流程：
   `300 rpm 调好 → 400 rpm 复核 → 600 rpm 复核 → 目标速度验收`。每升一级若波形变差，`Kp × 0.8` 再重跑。
2. **当前速度命令硬编码在** [cpu0_main.c:57](../user/cpu0_main.c#L57)（`motor_set_speed(400)`）。要改速度只有两条路：
   - **方案 A**：改源码重新烧录（每次换速度都要烧）。
   - **方案 B（推荐）**：参照 [uart_app.c:116-175](../code/app/uart_app.c#L116-L175) `change-xxx_pid` 的写法，自行新增 `speed-<value>` 串口命令，调试阶段效率差距是数量级的。
3. **横滚保护阈值是速度无关的常量**：超 21° 切电机、10° 恢复（[motor_app.h:18-19](../code/app/motor_app.h#L18-L19)）。即使低速，横滚过大依旧会触发保护，导致波形中段断开——**这不是 PID 问题**。
4. **速度变化后必须 `pid_reset`**（通过任意 `change-xxx_pid` 命令触发即可，[uart_app.c:108](../code/app/uart_app.c#L108)），否则新速度下积分项还背着旧速度的误差记忆。

**AI 诊断规则**：如果用户发来的波形不佳，AI 应在【现状分析】中**主动问清当前速度**——速度未知，很多症状无法准确归因。

### 8.2 内环 `gyro_pid`（角速度环）— 最重要

**目标**：CH2 (`gyro_y_rate`) 紧跟 CH1 (`target_gyro_rate`)，响应快、无振荡、不饱和。

**步骤**：

1. 把 `angle_pid.Kp=0, Kd=0`（或停用中环），手工给定 `target_gyro_rate` 阶跃（或让上层环传入方波）。
2. 从 `Kp=1.0` 起步，**倍增法**：1 → 2 → 4 → 直到 CH2 开始振荡。
3. 振荡边界 Kp 乘 **0.5~0.7** 作为工作值。
4. 几乎**不加 Ki**（角速度环有快速物理阻尼）。
5. 加极小量 `Kd` 抑制超调（通常 `Kd < 0.01 × Kp`），若噪声毛刺明显则不加。

**诊断看板**：

| CH3 (`error`) 形态 | 结论 | 调法 |
| ------------------- | ---- | ---- |
| 误差收敛慢、CH4 不饱和 | Kp 不够 | **Kp ×1.5** |
| CH2 高频抖动（啸叫） | Kp 过大 | **Kp ×0.7** |
| CH2 滞后明显 | Kd 或采样问题 | 小加 Kd（先 0.001 起） |
| CH4 贴 ±1500 | 饱和 | Kp 减小 / 检查舵机 / 降 `target_gyro_rate` 幅度 |

### 8.3 中环 `angle_pid`（角度环）

**目标**：CH5 (`pitch`) 稳定在 `target_angle`，无稳态偏差、无大超调。

**步骤**：

1. 内环先调好并锁定。
2. `Kp` 从 **10** 起步，倍增到车体出现低频摇摆。
3. 振荡边界 `Kp × 0.6` 作为工作值（默认 30 是经验起点）。
4. 如有稳态偏移（车总是歪一侧但不倒），加 **极小 Ki**（如 `0.05~0.2`），并依赖积分限幅 ±200 防止饱和。
5. `Kd` 用于抑制超调，典型 `0.5~2.0`。

**诊断看板**：

| CH5 (`pitch`) 形态 | 结论 | 调法 |
| ------------------ | ---- | ---- |
| 长期偏离 target_angle 但不发散 | 静差 | **+ Ki**（小步，0.05~0.2） |
| 来回慢摆（周期 >0.5s） | Kp 不够 或 Kd 过大 | Kp ↑ / Kd ↓ |
| 大幅超调后才稳下来 | Kd 不够 | **Kd ↑** |
| 高频振动（看起来像内环震，但周期略长） | Kp 太大 | **Kp ×0.7** |
| 向内环传的 CH1 忽大忽小跳变 | Kd 过大放大噪声 | **Kd ↓** |

### 8.4 外环 `steering_pid`（转向环）

**当前已接入固定输入试车**：在 [isr.c:81](../user/isr.c#L81) 直接调用 `balance_steering_loop(15.0f)`。

这说明当前工程已经在跑外环，但**现在喂的是固定误差 15°**，它只能用于粗试舵机与横滚响应，**不能代表真实的圆轨迹闭环控制**。

启用时：

- 输入可能是航向误差或循迹偏差。
- 输出被中环当作目标倾角（`target_angle`），限幅 ±30°。
- 调参思路：Kp 小步起（1.0），看 CH5 是否能跟上指令而不过调。
- 如果任务目标是"以固定半径绕圆"，AI 应优先建议用户打印 **第 6.1 节的外环专用数据**，而不是只看内环 5 通道。

### 8.5 调参前后必做

- **调参前**：`check-xxx_pid` 回读，AI 必须知道"改之前是什么值"。
- **调参后**：观察至少 3~5 个完整响应周期，再判断是否需要继续调。
- **改动幅度**：建议每次变化不超过 **±30%~50%**，避免越调越乱。

## 9. 常见症状 → 处方速查表

> 下表**供 AI 直接查表对应**。左列是用户描述 + 波形形态，右列是首选调整动作。

| 用户描述 / 波形症状 | 最可能原因 | 首选动作 |
| ------------------- | ---------- | -------- |
| "内环跟不动、CH2 远远落后 CH1" | gyro_pid.Kp 太小 | **Kp × 1.5~2** |
| "高频啸叫、舵机抖个不停" | gyro_pid.Kp 过大 或 Kd 过大 | **Kp × 0.7**，Kd 暂置 0 |
| "CH4 一直顶在 ±1500，车还是倒" | 内环饱和 / 机械限位不够 | Kp 降低，检查舵机与打角极限 |
| "车能站住但总往左/右歪不回" | angle_pid 静差 | **Ki += 0.05~0.1**（首次加 Ki 时） |
| "车来回慢摆（1~2 秒一周期）" | angle_pid.Kp 太小 或 Kd 过大 | Kp ↑ 30%，Kd ↓ 30% |
| "起步就大超调后才稳" | angle_pid.Kd 不够 | **Kd × 1.5** |
| "波形中段突然断了、电机也停了" | 横滚超 21° 触发保护 | **非 PID 问题**，先把车扶正/缓启，再调 Kp |
| "改完参数后短时间更不稳了" | pid_reset 清空了状态 | 等 2~3 秒再判断 |
| "CH5 和 CH1 完全无关系、像乱码" | IMU 或 pitch 方向不对 | **停止调参**，先核对 IMU 方向 |

【后续】

### 10.3 完整示例

**用户输入**：

> 我在调 angle 环。`check-angle_pid` 回读：`30.0,0.0,1.0,0.0,500.0`。
> 串口最后几行：
>
> ```
> 12.0,3.5,-8.5,-120,4.8
> 11.5,2.8,-8.7,-135,5.1
> 10.8,2.0,-8.8,-150,5.5
> ```
>
> 波形图：CH5 (`pitch`) 稳定在 5° 左右，不回归 0。

**AI 输出**：

```
【现状分析】
- CH5 (pitch) 稳定在 ≈5°，不回归到 target=0，呈现静差。
- CH1 (target_gyro_rate) 绝对值持续增大（-120→-150），说明中环一直在"喊"内环回正，
  但中环本身没有 I 项消静差能力。
- CH4 (servo_output) 未饱和，机械无故障。
- 判据：第 9 节「车能站住但总往左/右歪不回」→ angle_pid 静差，应首次引入 Ki。

【调整建议】
- angle_pid.Ki 由 0.0 调整为 0.10（首次加，保守起步）。
- Kp=30、Kd=1、target=0、limit=500 保持不变。
- 积分限幅 ±200 已在代码中生效（balance_app.c:35），无需改 limit。

【命令】
change-angle_pid(30.00,0.10,1.00,0.00,500.00)

【后续】
- 观察 5~10 秒，pitch 应逐步回到 0° 附近。
- 若仍有 1~2° 残余静差：Ki 提至 0.20。
- 若出现低频摆动（> 1 秒周期）：Ki 退回 0.05。
```
